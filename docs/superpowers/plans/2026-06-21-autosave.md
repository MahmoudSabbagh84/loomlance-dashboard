## Phase 1 — Shared infra + invoice-editor pilot

### 1a. `useAutosave` hook — `src/hooks/useAutosave.js`
- Inputs: `{ watch, getValues, trigger }` from react-hook-form, a `save(patch)` async fn, `debounceMs` (default 700), and a `enabled` flag (e.g. `status === 'draft'`).
- Behavior:
  - Subscribe to `watch`; on change, debounce (text) / flush-now (discrete fields — caller marks them).
  - `await trigger(changedFields)`; only include **valid** fields in the patch.
  - **Serialize**: a single in-flight promise; queue the latest patch; drop stale responses.
  - Track `status: 'idle' | 'saving' | 'saved' | 'error'`; expose `retry()`.
  - On success: call an optional `onSaved(patch)` for optimistic cache updates.
- Unit-test the reducer/queue logic (debounce, serialize, validation-gate, error→retry) with fake timers.

### 1b. `<SaveStatus>` — `src/components/ui/SaveStatus.jsx`
- Props: `status`, `onRetry`. Renders: "Saving…" (spinner), "Saved ✓" (fades), "Couldn't save — Retry" (error + button).
- Lives in the editor header where the Save button was.

### 1c. Invoice editor pilot — `src/features/invoices/InvoiceEditor.jsx`
- Remove the **Save** button + `onSubmit` toast flow.
- Wire header fields through `useAutosave` → `useUpdateInvoice` (patch).
- Wire line-items: on change (debounced) → `useReplaceLineItems`. Re-use existing mutations.
- `enabled = invoice.status === 'draft'`. For non-draft: render **read-only** (disabled inputs + a "Sent — read-only" badge; optional "Edit anyway" deferred).
- Optimistic `setQueryData(['invoices','detail',id])` + lazy invalidate `['invoices','list']`.
- Keep **Send / Download / status actions** exactly as explicit buttons.

### 1d. Verify
- `npm run lint`, `npx vitest run`, `npm run build`.
- Live (preview): create a `ZZ-` draft via the new picker → edit fields/line-items → confirm autosave persists (reload shows changes) → confirm "Saved" indicator → confirm a sent invoice is read-only. Clean up `ZZ-` rows (FK order).

**→ STOP. Review with user before Phase 2.**

## Phase 2 — Create/edit split for modal forms
For each of client / project / contract / expense / recurring:
- Split the modal into a **Create** flow (required fields → create row) and an **Edit** flow (autosaves).
- Edit modal: remove Save; wire fields through `useAutosave` → the existing update mutation; `<SaveStatus>` in the modal header; closing = done.
- Keep add/remove of sub-records (line items, contacts) and uploads explicit.

## Phase 3 — Profile/Payments + sweep
- Payments tab toggle + default instructions autosave (immediate on toggle, debounced on text). Logo upload stays explicit.
- Remove any remaining Save buttons; final lint/test/build; update `docs/phases.md`.

## Risks / watch-list
- **react-hook-form + autosave**: avoid save loops from `reset()`/default re-seeding; only save on user-originated changes (`formState.isDirty` + field-level dirty).
- **Line-items churn**: debounce + serialize so rapid row edits don't spam `replaceLineItems`.
- **Validation UX**: a required field cleared mid-edit must not wipe the server value — hold + error.
- **Read-only correctness**: ensure sent/paid invoices truly can't be mutated (defense in depth: RLS/RPC already restrict, but UI must reflect it).
- **Test data**: all live verification on `ZZ-` rows only; never against real seeded data.
