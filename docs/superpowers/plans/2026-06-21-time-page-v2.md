# Time Page v2 Implementation Plan

> **For workers:** Implement task-by-task. Steps use checkbox (`- [ ]`) syntax. Superpowers is NOT installed on this machine — execute inline; after each task run lint/tests/build and commit. Source design: `docs/superpowers/specs/2026-06-20-time-page-v2-design.md`. Origin QA: **F2**, **F3** in `docs/qa-findings.md`.

**Goal:** Make `/time` show *whose* work each entry is and turn billing into a one-click, **per-project** action — a "Ready to bill" panel (one row per project with unbilled time) replaces the "pick a client from all clients" modal. Time entries can be tagged to a **contract**, and contracts gain an optional **hourly rate** that pre-fills the entry rate.

**Architecture:** One Supabase migration adds `time_entries.contract_id` (FK `ON DELETE SET NULL` + index) and `contracts.hourly_rate`, creates the user-facing `SECURITY DEFINER` RPC `generate_invoice_from_time_for_project(p_project_id)` (groups line items by contract → rate), and drops the obsolete `generate_invoice_from_time(p_client_id)`. A pure helper in `src/lib/time.js` (unit-tested) does the per-project "ready to bill" aggregation. Thin API + TanStack-Query hooks back an updated `/time` page: a new `ReadyToBillPanel`, an updated `TimeEntriesTable` (Client/Contract columns), Client/Contract filters, and a contract picker + rate pre-fill in `TimerWidget` and `TimeEntryFormModal`. `ContractFormModal` gets an `hourly_rate` input. The old `GenerateInvoiceModal` is removed.

**Tech Stack:** React + Vite, react-hook-form, TanStack Query, Sonner, Tailwind `@/components/ui/*`, Supabase JS, Vitest.

## Global Constraints

- **Tier gate:** time tracking is `FEATURES.TIME_TRACKING` (tier_1+) — already gated on `/time` and in `TimerWidget`. Don't change gating.
- **Billing unit = project.** One invoice per project per generation, always `status='draft'`, never auto-sent. Generation is one-click → land on the existing invoice editor (the editor is the preview).
- **Column names (exact):** `time_entries` uses `hourly_rate`, `duration_minutes`, `ended_at`, `invoiced_on_invoice_id`, `billable`. "Completed" = `ended_at is not null`. "Unbilled" = `invoiced_on_invoice_id is null`.
- **RPC security:** `generate_invoice_from_time_for_project` is `SECURITY DEFINER`, `set search_path = public`, revoked from `public, anon`, granted to `authenticated`; verifies the project belongs to `auth.uid()`; raises `NO_UNBILLED_TIME` / `UNAUTHORIZED` with `errcode='P0001'` (already mapped in `src/lib/errors.js`). Reuse `next_invoice_number(v_user)` and `profiles.default_currency` exactly as the old RPC did.
- **FK on delete:** `time_entries.contract_id` = `ON DELETE SET NULL` (deleting a contract preserves time history; entry becomes untagged).
- **Contract picker scope:** contracts where `client_id = <project's client>` AND (`project_id = <chosen project>` OR `project_id IS NULL`) AND `status IN ('active','draft')`.
- **Rate pre-fill:** selecting a contract with a non-null `hourly_rate` pre-fills the entry rate (overrides profile default); user-editable; clearing the contract leaves the rate as-is; existing entries untouched.
- **Controlled selects:** all client/project/contract `<Select>`s must be controlled (`value` + `setValue`), per F11/F15 — options load async.
- **Reuse, don't reinvent:** invoice creation is server-side in the RPC; currency from `profiles.default_currency`; errors via `mapPostgresError`; money formatting via `@/lib/currency`; duration via `formatDuration`/`hoursFromMinutes` in `@/lib/time`.
- **Migrations:** apply via Supabase MCP `apply_migration` AND write the file to `supabase/migrations/<YYYYMMDDHHMMSS>_time_v2.sql`.
- **Commit style:** Conventional Commits, scope `time` (e.g. `feat(time): ...`). One commit per task. Run `npm run lint && npx vitest run && npm run build` before each commit.

## Task 1 — Migration: schema + RPC

- [ ] Add `alter table public.time_entries add column contract_id uuid references public.contracts(id) on delete set null;` + `create index on public.time_entries (contract_id);`
- [ ] Add `alter table public.contracts add column hourly_rate numeric;` (nullable; no constraint change to `value`).
- [ ] Create `generate_invoice_from_time_for_project(p_project_id uuid) returns uuid` — modeled on the existing `generate_invoice_from_time` (read it first):
  - `v_user := auth.uid()`; raise `UNAUTHORIZED` if null.
  - Load the project (`id = p_project_id and user_id = v_user`); raise `UNAUTHORIZED` if not found; capture `client_id`.
  - Count eligible entries (`user_id = v_user and project_id = p_project_id and billable and invoiced_on_invoice_id is null and ended_at is not null`); raise `NO_UNBILLED_TIME` if 0.
  - Insert draft invoice with `client_id = <project's client>`, **`project_id = p_project_id`**, `currency = coalesce(profiles.default_currency,'USD')`, `invoice_number = next_invoice_number(v_user)`, dates `current_date` / `current_date + 30`.
  - Insert line items grouped by `(contract_id, hourly_rate)`: `description = coalesce(c.title, p.name || ' — time')`, `quantity = round(sum(duration_minutes)::numeric/60, 2)`, `unit_price = coalesce(hourly_rate,0)`, `tax_rate=0, discount_rate=0`, `position = row_number() over (order by c.title nulls last, hourly_rate) - 1`. Left join `contracts c on c.id = te.contract_id`.
  - Stamp `invoiced_on_invoice_id = v_invoice_id` on those entries.
  - `return v_invoice_id`.
- [ ] `revoke all on function public.generate_invoice_from_time_for_project(uuid) from public, anon; grant execute to authenticated;`
- [ ] `drop function if exists public.generate_invoice_from_time(uuid);`
- [ ] Apply via MCP `apply_migration` AND save the file. Verify via MCP that the column/RPC exist.
