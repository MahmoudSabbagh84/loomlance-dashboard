## Fixed during this audit

After LOO-54 relabeled that StatsRow card "Overdue", the number was wrong (showed all open invoices). **Fixed:** now filters to `status==='overdue'` or sent/viewed past `due_date`. ## Systemic findings (the high-leverage fixes)

### P1 — Overlay accessibility: no focus trap / restore / consistent ARIA → LOO-69
Every overlay re-implements open/Escape/outside-click but **none trap focus, restore focus to the trigger, or share ARIA**: `Modal`, `Drawer`, `MobileNav`, `CommandPalette`, `NotificationBell`, the Topbar profile menu, the `TimerWidget` popover, `ColumnSettingsMenu`. Keyboard/SR users Tab out of dialogs into the inert page behind. Title-less `Modal` also has no accessible name. **One shared `useFocusTrap` + accessible overlay/menu primitive fixes all of them.** Fold in: the remaining sub-AA `fg-subtle` on interactive icon controls (close-X in Modal/Drawer/MobileNav/TagInput, TimerWidget discard-X, LoginPage password-eye) — WCAG 1.4.11. WCAG 2.4.3 / 2.1.2 / 4.1.2.

### P1 — Status messages / live regions missing → LOO-70
Dynamically-updating UI is silent to assistive tech: `SaveStatus` (autosave saving/saved/error — and the whole editor is autosave-only), `FieldError` (validation, no `role="alert"`, no `aria-invalid`/`aria-describedby` wiring), and the kanban drag (no dnd `announcements`). WCAG 4.1.3 / 3.3.1. The BrandingTab accent input already does `aria-invalid` — generalize that pattern.

### P1 — Charts have no text/table alternative or ARIA → LOO-71
All Recharts SVGs (`ReportChart`, dashboard `RevenueChart`, every report) are invisible to AT — no `role="img"`/`aria-label`. The data already exists in adjacent `<Table>`s / TopClients list; just wire `aria-hidden` on the chart + `aria-describedby` to the table (or `role="img"` + summary label). Also: `TimeReport` non-billable bar uses the sub-AA `fg-subtle` token as a chart fill (WCAG 1.4.11 graphical-object contrast). WCAG 1.1.1 / 1.3.1.

### P2 — Control semantics & selected-state are color-only → LOO-72
Reports tabs, both currency-pill implementations, and the SubscriptionTab segmented toggle convey "selected" via `bg-primary/10` only, with no `aria-selected`/`aria-pressed`/`role="radiogroup"`. `NotificationBell` advertises `aria-haspopup="menu"` but the popover isn't a menu. Extract a shared `SegmentedControl`/`Tabs` role pattern (ties to LOO-67 / LOO-59).

### P2 — Wide tables only horizontal-scroll on mobile → LOO-72
Time (9-col), Expenses (7-col), Aging/P&L (5-col), and the invoice-document tables overflow with no column-priority (`hidden md:table-cell`) or card fallback; core columns (duration/status/total) scroll off-screen. WCAG 1.4.10.

### P2 — "Display a count" = "fetch the whole list" (N+1) → LOO-65 (already filed)
`ProjectCard` runs `useTasks(project.id)` per card to show `tasks.length`; a 20-project grid fires 20 list queries (repeated in ProjectsTab). Use an embedded `tasks(count)` on the projects query.

## Positives (keep doing)
- **Theming is exemplary** — every surface consumes semantic tokens via Tailwind; the only literals are intentional scrims (`bg-black/60`), status-fill `text-white`, user-chosen project/label colors, and the PDF print palette. Dark mode "just works."
- **Lazy-loading discipline** — Recharts and react-pdf are dynamically imported everywhere, kept out of the main bundle.
- **Global `*:focus-visible` ring** at the root — consistent focus indication on every control.
- **Disciplined z-index scale** (10/20/30/40/50) — no arbitrary values.
- **Status never by color alone** — badges pair tint + text label throughout.
- **Real empty/loading states** with tailored copy across every surface.
- **Hooks/a11y discipline in feature code** — `aria-label` on icon buttons, `ConfirmDialog` for all destructive actions, correct rules-of-hooks around tier gates.
